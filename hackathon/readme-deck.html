Sentinel01 / 12
one-liner

CAW 治理的自主交易执行代理

Sentinel 把自然语言交易意图转成受限资金操作，由 Sentinel 做执行前风险判断，由 CAW Pact 保持最终执行边界。

project deckCAW-governed autonomous trading execution agent
Problem02 / 12
AI controls funds

AI Agent 直接控制钱包，风险不是“体验问题”，是资金边界问题。

  • Prompt injection 可能诱导模型绕过安全规则。
  • 模型可能幻觉出错误金额、错误合约或错误收款方。
  • 无限授权和不安全 swap 会放大损失。
  • 没有审计链路，事后无法解释每一步为什么发生。
核心风险AI 判断不能等于最终资金执行权
Core Idea03 / 12
final authority

AI 可以提出操作，Sentinel 可以批准，但最终执行权必须在 CAW Pact。

Agent 提出操作 自然语言 intent 被解析为结构化交易 proposal。
Sentinel 判断风险 硬规则、Agent review、记忆异常共同决定 execute / reject / confirm。
CAW Pact 兜底 即使 Sentinel 通过，超出 Pact 的操作仍会被拒绝。
设计原则Sentinel approved ≠ CAW must execute
Why CAW04 / 12
Cobo Agentic Wallet

CAW 不是装饰，它是资金执行和策略强制层。

WalletCAW 持有测试网资金，不把私钥交给 Agent。
Pact限制金额、策略、执行范围。
Policy Deny超出 Pact 时基础设施直接拒绝。
EvidenceCAW transaction / tx hash 可进入 audit trail。
关键差异不是普通 EOA，不是 mock wallet

Audit Trail08 / 12
explainable execution

每一次执行、拒绝、确认，都进入可展开的审计链路。

Decision ChainAgent A proposal、5 个硬规则、Agent B/C review、最终决策。
Execution EvidenceCAW wallet、Pact ID、request id、CAW transaction id、tx hash。
Sensitive RedactionAPI key、Pact credential、authorization header 不写入 audit。
评委可以从首页进入结果，也可以在 /audit 页面复盘每条记录。
审计explainable, replayable, redact sensitive fields
Agent Evidence09 / 12
not just if-else

Agent 能力作为证据层，不抢 CAW 主线。

  • Tool-use：合约验证、gas、token price 等只读工具证据。
  • Memory anomaly：结合用户历史交易模式，异常时提升到 confirm。
  • Agentic retry：风险过高时有限重提案，不做无限自治。
Agent 证据bounded, explainable, CAW-first

Boundary11 / 12
security boundary

这是黑客松原型，不是生产托管系统。

  • 演示范围是 Sepolia testnet 上的受限资金操作。
  • 不是 mainnet trading product，不承诺收益或生产级 custody。
  • 生产化需要正式安全审计、更多模拟、严格 policy templates、监控和运维控制。
  • 当前 async recovery 仍需后续增强：半程 wrap/approve 不能伪装成完整 swap 成功。
边界prototype / reference implementation
